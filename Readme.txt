Pet-project - Library
Web application for book accounting in library.
- Registering users
- Registering books
- Taking and getting back books by users

Role model:
- Users:
	* Getting information about books
	* Searching books by name
- Administators:
	* Registering and modifying list of books
	* Registering and modifying list of users
	* Giving books to users. Getting books back to library

Database structure:
- table "person"
	
	CREATE TABLE IF NOT EXISTS public.person
	(
		id integer NOT NULL GENERATED BY DEFAULT AS IDENTITY ( INCREMENT 1 START 1 MINVALUE 1 MAXVALUE 2147483647 CACHE 1 ),
		name character varying COLLATE pg_catalog."default" NOT NULL,
		year integer NOT NULL,
		password character varying COLLATE pg_catalog."default" DEFAULT 'password'::character varying,
		role character varying COLLATE pg_catalog."default",
		CONSTRAINT person_pkey PRIMARY KEY (id),
		CONSTRAINT person_name_key UNIQUE (name),
		CONSTRAINT person_name_key1 UNIQUE (name),
		CONSTRAINT person_year_check CHECK (year > 1900)
	)

- table "book"

	CREATE TABLE IF NOT EXISTS public.book
	(
		id integer NOT NULL GENERATED BY DEFAULT AS IDENTITY ( INCREMENT 1 START 1 MINVALUE 1 MAXVALUE 2147483647 CACHE 1 ),
		person_id integer,
		name character varying COLLATE pg_catalog."default" NOT NULL,
		author character varying COLLATE pg_catalog."default" NOT NULL,
		year integer,
		"time" timestamp without time zone,
		CONSTRAINT book_pkey PRIMARY KEY (id),
		CONSTRAINT book_name_key UNIQUE (name),
		CONSTRAINT book_person_id_fkey FOREIGN KEY (person_id)
			REFERENCES public.person (id) MATCH SIMPLE
			ON UPDATE NO ACTION
			ON DELETE SET NULL
	)
